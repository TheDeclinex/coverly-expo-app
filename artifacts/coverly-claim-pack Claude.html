Design Recommendations & Critique

What to put on page 1

The cover must answer three questions in under 5 seconds: Who is this for? What happened? What is the total? The current mockup achieves this — property address, event type badge, and reference number are all above the fold. The total claim value should optionally appear on the cover for quick assessor triage. Consider adding a QR code linking to a digital version.

Room vs item grouping

Room-level summary first, item detail on demand is the right hierarchy. Assessors navigate by room (they visit the property that way). Items within rooms should be sortable by value descending in a digital version — most expensive items deserve the most scrutiny. Move pantry goods, cleaning supplies, and other consumables to a "minor items" sub-table inside each room to reduce visual noise.

Summary metrics

The 4-card + 3-card metric layout works well. Key additions for V1: % of items with evidence (not just count), and a cover position indicator (under / over insured) as a traffic-light. These two metrics are the first things an assessor needs to know.

✅ Must print cleanly

  • Cover page (full colour with teal band)
  • Executive summary metrics
  • Room-by-room tables
  • Item schedule (all items)
  • Missing info checklist
  • Declaration page (with signature lines)

🔗 Clickable in digital PDF

  • Evidence reference numbers → jump to appendix
  • Item #IDs in tables → jump to item card
  • Room names in summary → jump to room section
  • Table of contents (add in V1)
  • Coverly URL on footer

📦 Move to appendix

  • Individual evidence thumbnails (bulk photos)
  • Full receipt PDFs
  • Warranty documents
  • Valuation certificates
  • Notes / internal fields

🗂 V1 essential fields

  • Item name, category, room
  • Brand, model (where known)
  • Quantity
  • Replacement value + basis
  • Evidence type (photo / receipt / none)
  • Review status (ready / flagged)

⏳ Add post-V1

  • Serial numbers
  • Independent valuation flag
  • Depreciation notes
  • Assessor override fields
  • Multi-property support
  • Barcode / product ID links

⚠️ Top design risks

  • Item tables go wide — cap at 6 columns for print
  • Badges lose meaning if overused — max 3 variants
  • Evidence thumbnails must be real photos in production
  • AI confidence labels need plain-English framing
  • Declaration wording needs NZ lawyer sign-off

Critique of this mockup

The teal brand application is strong and consistent. The page header / footer system creates good visual rhythm across a long document. The item card format is the right pattern — compact, scannable, badge-driven — but in production the two-column grid inside each card may need to collapse to single column for narrower items or print.

The biggest V1 gap in this mockup: there is no table of contents. A 10+ page document without TOC is a significant UX failure for assessors. Add a TOC as page 2 (push executive summary to page 3) with page numbers and clickable anchors. Second gap: the missing evidence placeholders (dashed boxes) should in production show actual photo thumbnails — the value of Coverly's claim pack lives entirely in the quality of the visual evidence.

Formatting recommendations

Use A4 paper size as the production target (not US Letter). Font stack: Inter or DM Sans for UI/labels, Georgia or Lora for body paragraphs. Keep teal to structural elements (header bands, section numbers, metric card accent) — not inline text — to preserve readability at low contrast on printed copies. Minimum body font 11pt for print; 13–14px for screen.
