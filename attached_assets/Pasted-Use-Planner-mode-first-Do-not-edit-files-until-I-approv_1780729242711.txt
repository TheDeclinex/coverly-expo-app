SCAN MODES

The mobile scan screen should support these 4 Coverly scan options:

1. Single photo room scan

* User selects/takes one room photo.
* AI detects multiple visible items.
* Results go to review screen.
* User saves selected items into selected room.
* This is the highest priority.

2. Single item scan

* User selects/takes one close-up item photo.
* AI returns one richer item result with brand/model/condition/value where possible.
* Results go to review screen.
* User saves item into selected room.
* This is second priority.

3. Multi-photo room scan

* User selects/takes up to 5 photos for the same room.
* For V1, plan either:

  * wire it if the existing UI Bakery/backend logic is clear, or
  * keep UI ready and return “coming soon”/not configured safely
* Later logic should detect exhaustively per photo and merge only when clearly the same object.

4. Video room scan

* Keep as placeholder/coming soon for now.
* Do not build video frame extraction in this tranche.

SCAN FLOW

Plan the scan flow as:

1. User chooses scan mode.
2. User selects property/file and room, or these are preselected from route params.
3. User captures/selects image(s).
4. App uploads/stages image(s) as needed.
5. App calls secure AI scan backend.
6. App shows a review screen.
7. User can edit detected item fields before saving.
8. User saves selected items.
9. Saved items use the existing item insert helper, not a separate insert path.
10. Room list refreshes and new items appear.
