Issues to fix:

1. Property and room cover images
   Newly created properties and rooms currently show blank/empty cover areas.

* If a property has no real cover photo, show a polished default property/home placeholder icon.
* If a room has no real cover photo, show a polished default room placeholder icon.
* Use suitable defaults, for example:

  * Property/home: home icon
  * Lounge/living room: sofa or room icon if available
  * Bedroom: bed icon if available
  * Kitchen: kitchen/utensils icon if available
  * Bathroom: bath icon if available
  * Generic room fallback: room/grid/home icon
* Do not leave large empty pale blocks with only a tiny icon unless intentionally styled.
* Add a user option to set or replace a real photo for a property and for a room.
* The user should be able to take a photo or upload/select from device gallery, using the existing Expo/image picker pattern if already present.
* Store the image in the existing Supabase storage convention if one already exists.
* Use existing database columns for cover images if they already exist.
* If property/room cover image columns do not exist, do not invent silently. Add a clear migration/SQL file and explain what column is needed, using existing naming conventions.

2. Item thumbnail expansion opens black overlay with no image
   When tapping an item thumbnail, the app opens a black overlay but does not show the image.

Required behaviour:

* Tapping an item thumbnail should open the real item/source image in the image viewer.
* If the item has `photoUrl`, `imageUrl`, `photo_url`, `image_url`, `source_image_url`, or similar, normalize and pass the correct URI into the modal.
* Reuse the same image URL logic that works elsewhere in the app.
* Do not open the black overlay if there is no valid image. Show a friendly fallback instead.
* Add loading/error states for the expanded image so failed image loads are visible and diagnosable.
* Make sure Supabase storage URLs are converted to usable public/signed URLs using the existing helper pattern.
* Avoid duplicating image URL logic across screens if a shared helper already exists.

Required behaviour:

* Pin placement must scale correctly to the rendered image size.
