Goal:
Add scan review pins and expandable image thumbnails across the Coverly mobile app, aligned with existing production schema and naming conventions.

Current state:

* AI scan works.
* Scan review returns detected items.
* Save one and Save all work.
* Saved scan-created items appear in the correct room.
* Scan-created items currently do not reliably save/show photos.
* Pins are not shown on the scan review screen.
* Thumbnails are not expandable across screens.

Production conventions:

* Edge Function name: scan-room-photo
* Storage bucket: inventory-photos
* Item image fields: inventory_items.photo_url and inventory_items.image_url
* Item pin field: inventory_items.image_pin jsonb
* Do not create new DB columns.
* Do not create new bucket names.
* Do not use item-photos.
* Do not use coverly-scan.
* Do not store base64 or local file:// URIs in the database.
* Do not change Add/Edit/Move/RLS logic unless required for shared image viewing only.

* Show the scan source image near the top of the review screen or inside an expandable preview.
* Overlay numbered pins for detected items where coordinates exist.
* Tapping a pin should scroll/focus/highlight the matching detected item card.
* Tapping a detected item card should highlight the matching pin.
* Use Coverly styling: dark green/teal pins, white number, subtle shadow.
* Keep layout usable on mobile.

Coordinate handling:

* If scan-room-photo returns valid coordinates, map them to inventory_items.image_pin.
* If no coordinates are returned, do not fake coordinates silently.
* Plan a manual tap-to-place pin fallback:

  * user taps image while an item is selected
  * save relative x/y coordinates, not pixel coordinates
  * e.g. image_pin = { x: 0.42, y: 0.61, sourcePhotoIndex: 0, type: "manual" }
* If manual pinning is too much for the first execution pass, plan it clearly as Phase 2 and show “No pin yet” safely.

3. Save pin data

* Do not save invalid strings or malformed objects.
* Do not break save-one or save-all.

4. Scan-created item images

* single photo room scan: V1 can use the room photo as the image for each detected item
* multi-photo scan: use sourcePhotoIndex if available; otherwise fallback to first selected image with a TODO
* avoid duplicate uploads in Save all by uploading each source image once and reusing the result
