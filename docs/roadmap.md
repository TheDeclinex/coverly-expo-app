# Coverly Roadmap

## Near-term priorities

### 1. Stabilise Expo app

Focus:
- Image reliability.
- Pins.
- Thumbnails.
- Room/item save flows.
- Property/room cover images.
- Navigation and empty states.

### 2. Improve onboarding and first-use guidance

Focus:
- Ask for property name.
- Ask for insured contents value.
- Guide user to add rooms.
- Guide user to first scan.
- Reduce blank/directionless screens.

### 3. Replacement pricing module

Focus:
- Replicate strongest UI Bakery behaviours.
- Listing cards with images.
- Filter pills by value bands.
- “Use this listing”.
- Entitlement checks.

### 4. Native billing

Focus:
- RevenueCat setup.
- Native purchase flow.
- Entitlement sync to Supabase.
- Clear upgrade UI.

### 5. Claim packs

Focus:
- Room/item selection.
- PDF layout.
- Evidence appendix.
- Subscriber and one-off access rules.

### 6. Dev/prod environment

Focus:
- Separate dev/prod config.
- Safer promotion path.
- GitHub branch discipline.
- Avoid accidental production data mutation.

## Validation priorities

Product/business validation should run alongside engineering:

- Do households care enough to document contents?
- Is “contents value vs insured value” compelling?
- Are people willing to scan their home?
- Are claim packs valuable before a claim?
- What do insurance professionals see as credible/useful?
- Which pricing model feels acceptable?

## Known non-urgent/future ideas

- Assessor/professional mode.
- Multi-user family access.
- Receipt/evidence automation.
- Barcode verification.
- Voice command.
- International pricing/localisation.
- Broker/insurer partnerships.
